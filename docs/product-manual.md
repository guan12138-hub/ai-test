# AI智能食材管理系统 - 产品说明书

## 一、项目概述

### 1.1 项目背景
随着餐饮行业数字化转型加速，食堂/餐饮门店在日常运营中面临食材损耗高、库存管理混乱、采购计划不科学等问题。本系统旨在通过AI技术赋能食材全生命周期管理，实现智能化库存管控、AI辅助采购决策、智能菜品搭配推荐等功能，降低食材损耗，提升运营效率。

### 1.2 项目定位
面向食堂/餐饮门店/家庭用户的**食材全生命周期管理工具**：
- **Web端**：用于电脑批量管理库存、采购、报表分析
- **小程序端**：用于手机快速录入食材、扫码查保质期、随手生成菜谱

### 1.3 目标用户
- 食堂管理人员
- 餐饮门店店主/厨师
- 家庭用户

## 二、系统功能架构

### 2.1 基础业务功能

#### 2.1.1 用户管理
- 用户注册/登录（JWT认证）
- 角色权限区分：管理员(ADMIN)、普通用户(USER)
- 用户信息管理

#### 2.1.2 食材管理
- 食材入库：名称、分类、条码、库存量、单价、生产日期、过期日期、储存方式
- 食材出库：消耗记录、损耗登记
- 库存台账：实时查看库存状态
- 保质期自动预警：7天内过期自动标记预警

#### 2.1.3 食材分类管理
- 多级分类：蔬菜、水果、肉类、禽类、水产、蛋奶、调味品、粮食、冻品等
- 分类增删改查

#### 2.1.4 供应商管理
- 供应商信息管理：名称、联系人、电话、地址
- 供应商增删改查

#### 2.1.5 采购订单管理
- 采购订单创建、编辑、删除
- 订单状态追踪：待处理、已完成、已取消
- 关联供应商和食材

#### 2.1.6 数据报表
- 食材分类分布图表
- 月度消耗趋势图表
- 库存状态统计
- 工作台仪表盘

### 2.2 AI智能功能（四大核心模块）

#### 2.2.1 AI问答助手/智能客服
- **功能描述**：对接Coze平台AI机器人，实现食材知识咨询、库存管理答疑、菜品搭配问答、用户售后咨询
- **技术实现**：通过Coze API创建专属食材管理AI机器人工作流，支持多轮对话
- **应用场景**：用户可询问食材储存方法、菜品搭配建议、库存问题等

#### 2.2.2 AI文章生成
- **功能描述**：自动生成菜谱文案、食材采购计划表、食堂菜单方案、食材存储科普短文、食材损耗分析报告
- **技术实现**：通过Coze API调用大语言模型，根据提示词模板生成结构化内容
- **应用场景**：食堂每周菜单生成、采购计划自动制定

#### 2.2.3 AI绘图生成
- **功能描述**：生成菜品效果图、食材分类海报、食堂菜单配图、食材储存示意图、采购清单可视化图表
- **技术实现**：通过Coze API调用图像生成模型
- **应用场景**：菜单配图、宣传海报制作

#### 2.2.4 AI智能分析
- **功能描述**：食材库存损耗数据分析、食材采购成本分析、食材保质期预警分析、用户食材消耗习惯文本数据情感/统计分析
- **技术实现**：将数据文本传递给Coze API进行深度分析
- **应用场景**：月度损耗报告、采购成本优化建议

## 五、AI接口设计

### 5.1 Coze平台对接
本系统通过Coze平台创建专属食材管理AI机器人，提供以下API接口：

#### 5.1.1 智能对话
- 接口：POST /api/ai/chat
- 功能：用户发起对话，AI返回回复
- 请求参数：query(问题), userId
- 响应：response(AI回复内容)

#### 5.1.2 文章生成
- 接口：POST /api/ai/article
- 功能：根据主题和类型生成文章
- 请求参数：topic(主题), type(类型：RECIPE/PURCHASE_PLAN/MENU/STORAGE/ANALYSIS)
- 响应：content(文章内容)

#### 5.1.3 AI绘图
- 接口：POST /api/ai/image
- 功能：根据提示词生成图片
- 请求参数：prompt(图片描述)
- 响应：imageUrl(图片URL)

#### 5.1.4 智能分析
- 接口：POST /api/ai/analyze
- 功能：对食材数据进行分析
- 请求参数：dataType(分析类型), dataContext(数据内容)
- 响应：analysis(分析报告)
